fix(release): pass version to Inno Setup so installer filename matches tag

The iss file hardcoded MyAppVersion=0.4.2, so every release built an installer
named OpenBox-0.4.2-windows-amd64-setup.exe regardless of the actual tag.

Fix: wrap the version defines in #ifndef so they can be overridden from the
iscc command line via /D flags. release.yml now passes
/DMyAppVersion=<tag-version> /DMyAppVersionFull=<tag-version>.0 so the
installer filename always matches the release tag.

Also revert the broken v0.4.3 release (had wrong Windows installer name)
and will cut v0.4.4 with the fix.
